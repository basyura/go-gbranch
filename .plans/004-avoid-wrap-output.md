# 変更計画: 出力折り返し防止のための幅補正

## 目的
- Git Bash で出力が折り返されないように行幅を補正する
- 取得できた幅に収まるように表示を確実に切り詰める

## 方針
- 端末幅取得は既存のロジック（term.GetSize → COLUMNS）を維持する
- 取得した幅から 1 を減算して、折り返しを回避する
- 取得した幅に合わせて remote と message を確実に短縮する
- 既存の表示整形ルールは維持する

## 手順
1. getTerminalWidth の現状を確認する
2. 取得した端末幅に 1 文字分の補正を入れる
3. 取得した幅に合わせて出力を切り詰める処理を調整する
4. 変更後の出力サンプルを確認する

## 実装概要
- term.GetSize 失敗時は COLUMNS を確認する（従来通り）
- 取得した端末幅から 1 を減算して使用する
- 幅超過時の切り詰め処理は既存の remote → message の順序を維持する

## 影響範囲
- main_gbranch.go

## 確認事項
- 端末幅取得に失敗した場合は従来どおり制御なしで良いか
